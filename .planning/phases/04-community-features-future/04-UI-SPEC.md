| sm | 8px | Compact element spacing, badge gaps |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card internal spacing |

## Interaction Patterns

### Review Form

1. User clicks "Write a Review" -> ReviewForm expands inline (or scrolls to form if not visible)
2. Stars: hover previews fill color on stars up to hovered position. Click locks selection. Must select at least 1 star to submit.
3. Content textarea: optional, max 500 characters, shows character count
4. Platform selector: optional dropdown, populated from existing Platform list
5. UseCase: optional text input, single line
6. Submit button: disabled until rating selected. On submit, POST to API.
7. If user already has review: form pre-fills with existing data. Submit button says "Update Review".
8. After submit: form collapses, review appears in list with success toast "Review updated" / "Review submitted"

### Tag Voting

1. Tags displayed as pills in a flex-wrap grid
2. Each pill: label text + vote count in parentheses
3. Hover: subtle background shift + border accent
4. Click (unvoted): pill fills with accent color, count increments by 1, brief scale pulse (1.02x) via framer-motion
5. Click (voted): pill returns to default, count decrements, brief fade transition
6. If user already has 3 votes and clicks a 4th: toast message "You can vote for up to 3 tags per tool", no state change
7. If user is signed out and clicks any tag: show inline "Sign in to vote" prompt with GitHub login link
8. Tags sorted by vote count descending. Ties broken by alphabetical order.

### Tool Submission

1. Page: `/submit` (replaces existing placeholder)
2. Form in centered card (max-w-2xl, matching existing submit page layout)
3. repoUrl field: on blur, validate format. On valid format, show loading spinner. On API confirmation, show green checkmark with repo name. On failure, show red error.
4. Notes: optional textarea, max 300 characters
5. SuggestedTags: optional row of checkbox pills from preset tag list
6. Submit: disabled until repoUrl validated. On success, replace form with success card "Thank you! Your submission is pending review."
7. Signed-out users: entire form replaced with "Sign in to submit a tool" + GitHub login button

### Admin Review Page

1. Route: `/admin/submissions` (outside locale routing, server-rendered)
2. Simple table/list layout: each row shows repo name, submitter username, submitted date, status badge
3. Click row or "View" button to expand details: repo URL, notes, suggested tags, GitHub preview (stars, description, language)
4. Two action buttons: "Approve" (accent color) and "Reject" (destructive)
5. Reject: inline confirmation prompt appears below the row
6. After action: row animates out, status updates in list
7. Access: server-side auth check against ADMIN_GITHUB_IDS env var. Unauthorized users see 404.

### Login Button (Header)

1. Signed out: shows "Sign in" text + GitHub icon, rounded-lg button with border style matching locale toggle
2. Signed in: shows user avatar (24px round) + username text, no border
3. Click signed out: redirect to GitHub OAuth
4. Click signed in: dropdown with "Sign out" option (or just sign out directly)

## Accessibility Requirements

| Element | Requirement |
|---------|-------------|
| Star rating input | Role: radiogroup. Each star: role radio, aria-label "1 star" to "5 stars". Keyboard: arrow keys to change, Enter to select. Focus ring visible. |
| Tag voting pills | Role: button. aria-pressed="true/false". aria-label includes tag name + vote count. Keyboard: Enter/Space to toggle. |
| Review list | Each review in article element. Heading level for reviewer name. Time element with datetime attribute. |
| Rating distribution | Each bar: role meter or visually hidden text "5 stars: 72%, 92 reviews". Accessible name for each bar. |
| Login button | aria-label includes "Sign in with GitHub" or "Signed in as {username}". |
| Form validation errors | aria-live="polite" region for error messages. aria-invalid on invalid fields. aria-describedby linking to error text. |
| Submit form | Required field indicator (*), fieldset/legend for suggested tags group. |
| Admin actions | Confirm/cancel pattern keyboard accessible. Focus trapped in confirmation dialog. |
