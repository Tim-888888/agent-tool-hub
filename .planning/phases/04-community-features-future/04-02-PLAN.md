Create `src/app/api/tools/[slug]/reviews/route.ts` with two handlers:

**GET handler** -- List reviews for a tool:
1. Accept URL search params: `page` (default 1), `limit` (default 10, max 50), `sort` (default "newest")
2. Look up tool by slug. If not found, return `errorResponse("Tool not found", 404)`
3. Query reviews with `prisma.review.findMany` where toolId matches, include user relation (select id, name, image only)
4. Sort by createdAt desc for "newest"
5. Return `{ success: true, data: reviews, meta: { total, page, limit, totalPages } }`

**POST handler** -- Submit or update review (per D-05, D-06, D-07, D-08):
1. Call `requireAuth()` -- if error, return the 401 error response
2. Parse request body, validate with Zod:
   ```typescript
   const reviewSchema = z.object({
     rating: z.number().int().min(1).max(5),
     content: z.string().max(500).optional(),
     platform: z.string().max(50).optional(),
     useCase: z.string().max(100).optional(),
   })
   ```
3. Look up tool by slug. If not found, return 404
4. Upsert review (per D-07 -- one review per user per tool):
   ```typescript
   const review = await prisma.review.upsert({
     where: { userId_toolId: { userId: session.user.id, toolId: tool.id } },
     create: { userId: session.user.id, toolId: tool.id, rating, content, platform, useCase },
     update: { rating, content, platform, useCase },
   })
   ```
5. Recalculate avgRating and ratingCount (per D-08 -- real-time aggregation):
   ```typescript
   const stats = await prisma.review.aggregate({
     where: { toolId: tool.id },
     _avg: { rating: true },
     _count: true,
   })
   await prisma.tool.update({
     where: { id: tool.id },
     data: {
       avgRating: Math.round((stats._avg.rating ?? 0) * 10) / 10,
       ratingCount: stats._count,
     },
   })
   ```
6. Return successResponse with the review data

Follow existing route handler pattern from `src/app/api/tools/[slug]/route.ts`: use `await params` for dynamic segments, try/catch wrapper, errorResponse for errors.
