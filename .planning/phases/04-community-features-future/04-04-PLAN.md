Purpose: Enables community-driven tool discovery. Users submit GitHub repos, the system validates they exist and are not duplicates, admins review and approve, and approved submissions automatically become active tools with fetched metadata.

Output: Working submit flow (form -> validation -> pending confirmation), admin review page (list -> preview -> approve/reject), and auto-tool-creation on approval.

   ```typescript
   const existingTool = await prisma.tool.findFirst({ where: { repoUrl } })

   if (existingSubmission) return errorResponse("This repository has already been submitted and is pending review", 409)
   ```
5. Validate repo exists on GitHub per D-13:
   ```typescript
   try {
     await fetchRepoData(parsed.owner, parsed.repo)

   }
   ```
6. Create submission per D-17:
   ```typescript
   const submission = await prisma.submission.create({
   ```

**Step 2: Create src/app/api/admin/submissions/route.ts**

GET handler per D-16 (list PENDING submissions):
1. Call `requireAuth()` -- if error, return 401
2. Check `isAdmin(session.user.id)` -- if not, return 403
3. Query submissions:
   ```typescript
   const submissions = await prisma.submission.findMany({
     where: { status: "PENDING" },
     include: { user: { select: { name: true, image: true } } },
     orderBy: { createdAt: "desc" }
   })
   ```
4. Return `successResponse(submissions)`

PATCH handler per D-11 (approve/reject), D-15 (auto-fetch on approval):
1. Call `requireAuth()` -- if error, return 401
2. Check `isAdmin(session.user.id)` -- if not, return 403
3. Parse body with Zod:
   ```typescript
   const actionSchema = z.object({
     submissionId: z.string(),
     action: z.enum(["approve", "reject"]),
   })
   ```
4. Find submission by ID, verify status is PENDING -- if not found or already processed, return 404
5. If action is "reject":
   ```typescript
   await prisma.submission.update({
     where: { id: submissionId },
     data: { status: "REJECTED", reviewedAt: new Date() }
   })
   return successResponse({ status: "REJECTED" })
   ```
6. If action is "approve" per D-15 -- auto-fetch GitHub data:
   ```typescript
   const parsed = parseRepoUrl(submission.repoUrl)
   if (!parsed) return errorResponse("Invalid repo URL", 400)
   const { owner, repo } = parsed
   const [repoResult, readmeResult] = await Promise.allSettled([
     fetchRepoData(owner, repo),
     fetchReadme(owner, repo),
   ])
   const repoData = repoResult.status === "fulfilled" ? repoResult.value : null
   if (!repoData) return errorResponse("Cannot fetch repo data", 400)
   const readmeContent = readmeResult.status === "fulfilled" ? readmeResult.value : null
   const features = readmeContent ? extractFeatures(readmeContent) : []
   const installGuide = readmeContent ? extractInstallGuide(readmeContent) : null
   const score = computeScore({
     stars: repoData.stargazers_count,
     forks: repoData.forks_count,
     lastCommitAt: new Date(repoData.pushed_at),
     npmDownloads: null,
   })
   ```
   Then create Tool and update Submission in a transaction:
   ```typescript
   const tool = await prisma.tool.create({
     data: {
       slug: repo,
       name: repoData.name || repo,
       description: repoData.description ?? "",
       repoUrl: submission.repoUrl,
       type: "MCP_SERVER",
       stars: repoData.stargazers_count,
       forks: repoData.forks_count,
       openIssues: repoData.open_issues_count,
       language: repoData.language,
       license: repoData.license?.key ?? null,
       lastCommitAt: new Date(repoData.pushed_at),
       featuresEn: features.length > 0 ? features : [],
       installGuide: installGuide ? { markdown: installGuide } : null,
       status: "ACTIVE",
       score,
     }
   })
   await prisma.submission.update({
     where: { id: submissionId },
     data: { status: "APPROVED", reviewedAt: new Date(), toolId: tool.id }
   })
   return successResponse({ status: "APPROVED", toolId: tool.id })
   ```

IMPORTANT: The slug must be unique. If a tool with this slug already exists, append a numeric suffix or use owner-repo format.

```tsx
import { auth } from "@/auth"
import { redirect } from "next/navigation"
import { prisma } from "@/lib/db"

export default async function AdminSubmissionsPage() {
  const session = await auth()
  if (!session?.user?.id) redirect("/")
  const ADMIN_IDS = (process.env.ADMIN_GITHUB_IDS ?? "").split(",").filter(Boolean)
  if (!ADMIN_IDS.includes(session.user.id)) redirect("/")

  const submissions = await prisma.submission.findMany({
    where: { status: "PENDING" },
    include: { user: { select: { name: true, image: true } } },
    orderBy: { createdAt: "desc" },
  })

  return (
    <div className="mx-auto max-w-6xl px-4 py-12 sm:px-6 lg:px-8">
      <h1 className="text-3xl font-bold text-[var(--text-primary)]">
        Submissions ({submissions.length} pending)
      </h1>
      {/* Render submission list */}
      {submissions.length === 0 ? (
        <p className="mt-6 text-[var(--text-secondary)]">No pending submissions.</p>
      ) : (
        <div className="mt-6 space-y-4">
          {submissions.map(sub => (
            <div key={sub.id} className="rounded-2xl border border-[var(--border)] bg-[var(--bg-primary)] p-5">
              <div className="flex items-center justify-between">
                <div>
                  <p className="font-medium text-[var(--text-primary)]">{sub.repoUrl}</p>
                  <p className="text-sm text-[var(--text-secondary)]">
                    by {sub.user?.name ?? "Unknown"} on {sub.createdAt.toLocaleDateString()}
                  </p>
                </div>
                <span className="rounded-full bg-[var(--color-warning)]/10 border border-[var(--color-warning)]/30 px-2.5 py-1 text-xs font-medium text-[var(--color-warning)]">
                  PENDING
                </span>
              </div>
              {sub.notes && (
                <p className="mt-3 text-sm text-[var(--text-secondary)]">{sub.notes}</p>
              )}
              {sub.suggestedTags.length > 0 && (
                <div className="mt-2 flex flex-wrap gap-1">
                  {sub.suggestedTags.map(tag => (
                    <span key={tag} className="rounded-lg bg-[var(--bg-tertiary)] px-2 py-0.5 text-xs text-[var(--text-secondary)]">{tag}</span>
                  ))}
                </div>
              )}
              <form action={async () => {
                "use server"
                // Server action for approve/reject
              }} className="mt-4 flex gap-3">
                {/* Buttons rendered here, actions handled via server actions or client fetch */}
              </form>
            </div>
          ))}
        </div>
      )}
    </div>
  )
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Submit API | Untrusted repoUrl, notes, suggestedTags |
| Submit API -> GitHub | External API call to validate repo |
| Admin API -> Database | Privileged mutation: create Tool, update Submission status |
| Admin page | Only ADMIN_GITHUB_IDS users should access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | T | POST /api/submit | mitigate | Zod schema validation; repoUrl validated via parseRepoUrl (exact hostname check) + fetchRepoData (GitHub API 404 check) |
| T-04-14 | D | POST /api/submit | mitigate | Dedup check prevents repeated submissions; auth required limits abuse; GitHub API rate limit applies |
| T-04-15 | E | PATCH /api/admin/submissions | mitigate | requireAuth() + isAdmin() check; only ADMIN_GITHUB_IDS can approve/reject |
| T-04-16 | S | Admin page | mitigate | Server-side auth check; non-admins redirected to /; no client-side role check bypass possible |
| T-04-17 | D | GitHub API abuse via submission | accept | Uses existing GITHUB_TOKEN; unauthenticated fallback 60 req/hr; per-submission validation only |
</threat_model>
