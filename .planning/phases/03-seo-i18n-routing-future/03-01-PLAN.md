## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> proxy.ts | Untrusted Accept-Language header, arbitrary URL paths |
| proxy.ts -> [locale] segment | Rewritten/redirected URLs must only contain validated locales |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | proxy.ts | mitigate | Validate locale against hardcoded allowlist ['en', 'zh'] before constructing redirect/rewrite URLs. Never use user-supplied locale directly from query params. |
| T-03-02 | Spoofing | proxy.ts | mitigate | Only match exact locale segments from pathname. Open redirect prevention: do not allow locale to contain '/' or '..' characters. |
| T-03-03 | Tampering | [locale]/layout.tsx | mitigate | Call notFound() for invalid locale values. isValidLocale checks against exact string match. |
| T-03-04 | Information disclosure | proxy.ts | accept | No sensitive data in proxy logic. Locale preference is not PII. |
</threat_model>
