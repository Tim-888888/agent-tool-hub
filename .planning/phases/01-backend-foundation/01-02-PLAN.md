Create `src/lib/api-utils.ts` with shared helpers used by all API routes:

1. `parsePagination(searchParams: URLSearchParams)` — returns `{ page: number, limit: number, skip: number }`:
   - page: Math.max(1, parseInt(sp.get('page') ?? '1'))
   - limit: Math.min(100, Math.max(1, parseInt(sp.get('limit') ?? '20')))
   - skip: (page - 1) * limit

2. `buildWhereClause(searchParams: URLSearchParams)` — returns Prisma where object:
   - Default: `{ status: { in: ['ACTIVE', 'FEATURED'] } }` (only show active/featured tools)
   - If `q` param: add `OR: [{ name: { contains: q, mode: 'insensitive' } }, { description: { contains: q, mode: 'insensitive' } }, { descriptionZh: { contains: q, mode: 'insensitive' } }, { tags: { has: q } }]`
   - If `type` param: add `type: type.toUpperCase()` (convert 'mcp' to 'MCP_SERVER' via mapping: mcp -> MCP_SERVER, skill -> SKILL, rule -> RULE)
   - If `platform` param: add `platforms: { some: { platform: { slug: platform } } }`
   - If `category` param: add `categories: { some: { category: { slug: category } } }`

3. `buildOrderBy(sortParam: string | null)` — returns Prisma orderBy object:
   - 'recent' -> `{ lastCommitAt: 'desc' }`
   - 'rating' -> `{ avgRating: 'desc' }`
   - 'name' -> `{ name: 'asc' }`
   - default (null, 'stars', anything else) -> `{ stars: 'desc' }`

4. `mapToolResponse(tool: any)` — flattens Prisma join table shape to frontend shape:
   - Input: `{ categories: [{ toolId, categoryId, category: Category }], platforms: [{ toolId, platformId, platform: Platform }], ...toolFields }`
   - Output: `{ ...toolFields, categories: tool.categories.map(tc => tc.category), platforms: tool.platforms.map(tp => tp.platform) }`
   - Handle null/undefined categories and platforms arrays defensively

5. `TOOL_PRISMA_INCLUDE` — exported constant for the standard Prisma include object used in all tool queries:
   ```typescript
   export const TOOL_PRISMA_INCLUDE = {
     categories: { include: { category: true } },
     platforms: { include: { platform: true } },
   } as const;
   ```

6. `successResponse(data: any, meta?: object)` — wraps in `{ success: true, data, meta }` with status 200
7. `errorResponse(message: string, status: number)` — wraps in `{ success: false, error: message }` with given status

**1. `src/app/api/tools/route.ts`** — GET /api/tools (list with pagination, filtering, sorting):
```typescript
import { type NextRequest } from 'next/server';
import { prisma } from '@/lib/db';
import { parsePagination, buildWhereClause, buildOrderBy, mapToolResponse, TOOL_PRISMA_INCLUDE, successResponse, errorResponse } from '@/lib/api-utils';

export async function GET(request: NextRequest) {
  try {
    const sp = request.nextUrl.searchParams;
    const { page, limit, skip } = parsePagination(sp);
    const where = buildWhereClause(sp);
    const orderBy = buildOrderBy(sp.get('sort'));

    const [tools, total] = await Promise.all([
      prisma.tool.findMany({ where, orderBy, skip, take: limit, include: TOOL_PRISMA_INCLUDE }),
      prisma.tool.count({ where }),
    ]);

    return successResponse(tools.map(mapToolResponse), {
      total, page, limit, totalPages: Math.ceil(total / limit),
    });
  } catch (error) {
    return errorResponse('Failed to fetch tools', 500);
  }
}
```
